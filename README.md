# 自动化排课系统

这是一个基于 Python/PyQt5 构建的高性能自动化排课系统，专为解决高中（多班级、分主副科、密集型跨班交叉任课）场景下极端复杂的约束满足问题（CSP）。

## 核心架构：双阶段解耦引擎 (Two-Pass Scheduling Engine)
为了避免算力在全校园高密度的课程排列中陷入几何级图论回溯（即常说的“深渊死锁”），核心排课算法 `scheduler.py` 采用了“主干 - 边角”两段式剥离回溯策略。

### 阶段一：正课主干排布 (Phase 1: Standard Courses)
处理周一至周五常规时段（如每日常规的 8 节课）的主副科安排。
1. **逐班打包排布 (Class-by-Class Packing MRV)**：
   摒弃了“按全校同一科目推进”的极易引发远端回溯死锁的做法，引擎通过将变量排序策略定为 `根据班级聚类 -> 连排优先(数学等) -> 课时高者优先`，确保在任何时间尝试安插某班课程时，该班物理网格处于大量空置的纯净状态，以最少的回溯成本实现局部最优。
2. **强制硬剪枝 (Max Occurrences Per Day Limit)**：
   自动识别课时总数，采用纯算术界限截断：`max_per_day = (总课时 + 天数 - 1) // 天数`。比如高中语文一周 6 节（非连排），系统严格截断该科在同班一天内的次数（最大 2 节），以逼迫算法自动把主课打平发散分布，无需增加人为软性调权。

3. **教师防疲劳控制 (Teacher Fatigue Balance)**：
   引入动态上限拦截模型。系统自动计算每位老师的“单日极限承载 = (周总课时 // 5天) + 2节缓冲”。在排课过程中实时监控跨班级总负载，强制打平教师分布，防止出现“周一满课、周二无课”的极端现象，保护教师精力及授课质量。

### 阶段二：早晚自习专项排布 (Phase 2: Duty/Study Sessions)
处理孤立且高排斥性的特殊排程（如 `早读甲/乙` 和 `晚修甲/乙`）。
1. **排他运行域**：
   自习完全不占用正课池区域（排布于 `0` 与 `9~11` 号特供槽），故阶段一完成后可以直接冻结基座，将极度碎片化的早晚自习任务抽离至此阶段集中运算。
2. **非饱和匹配策略**：
   在全校早/晚排班中，绝对满载的 Bipartite Matching（100%匹配网格）极易产生死局。通过保留 15%-30% 的非定宽空间（如晚修3设为无监督自修，或早读预留缓冲带节数），完美规避算法末端的组合爆炸。

## 后置负载统计与核查 (Post-Scheduling Analytics)
系统在解算完成后，提供了直观的**“教师排课负载统计”**工具：
1. **全维度覆盖**：自动汇总并对比每位教师在“正课”、“早读”及“晚修”三个维度的实际排课节数。
2. **可视化平衡校验**：支持交互式表头排序，管理人员可以一键找出全校工作量最重或最轻的老师，作为手动微调或未来教务分配的量化参考。
3. **科目透视**：统计表内嵌科目归属，方便查看同教研组内的任务分配公平性。

## 高灵活性数据备份与迁移 (Flexible Configuration Backup)
在“数据管理”面板中，系统提供了一键 **“导出当前配置”** 的功能：
1. **所见即所得的数据抓取**：当点击导出时，即使界面上的表格处于草稿状态（即尚未点击“保存”录入内核），系统依然会在毫秒级内自动采集界面中所有的基础资料，并连同当前已生成的排课课表一起打包导出。
2. **自定义分发与归档**：支持自定义输出路径与配置名（如：`上学期课表.json`）。该设计使得系统的全量数据不仅具备本地存档属性，也可以随时以 `.json` 格式进行无缝迁移或在教务处同事间共享。

## 防呆拦截与规则校验体系
核心判断方法位于 `models.py -> check_slot_available()`。系统的配置与使用受到以下绝对规则锁定：
1. **教师多线冲突检查**：同一时刻，一位教师无论教授多少班级，其所占课槽不能有交叠。
2. **硬性不可排时间排查**：依据配置字符串（如 `周一第2节, 周三第4节`）精准排挤。
3. **授课年级（目标防线）拦截**：
   如果为排课指定了授课年级配置串（如 `高一,高二`）。在底层引擎运转或界面手动操作中，如强行绑定一位超范围教学的老师去不对应的班级（如指派去了高三），系统直接触发拦截报警。此设置既可兼容老数据，也可严格控制越级指派。

## 拟真推演结构 (`generate_large_data.py`)
系统内置了一个高度符合真实世界教务处的测试生成器：
- 主干科目采取单一年级制（一课多班），而小科目（如政史地）则采取双年级跨级任教（如仅教 `高一,高二`）。
- **早读防洪墙算法**：通过构建交替矩阵，强制所有班级的连续早读时间由 `语文` 和 `英语` 按黄金切分法平分，系统底层严格防范因基数错位导致的“连着几天只背诵一门副科”的灾难偏科事故。

3. **课程命名标准化**：测试数据及系统生成逻辑自动在课程名后附加年级前缀（如 `高一语文`），课程管理界面更加直观，避免多级混排时名称混淆。

## 联调建议
由于算法极致剪枝，全校 24 班、最高满载 400 + 时段网格的排布时间应能在 **2~5 秒** 之内推算成功。如果在测试或真实生成中卡死，**绝对不能盲目加大回溯上限 `max_iterations`**，而应该检查是否有教师课时超越了物理空间限制（即发生了无法解套的满载溢出），通过微调教师容量或削减几节课时即可再次获得秒级排配。
